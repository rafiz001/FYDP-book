\section{Environment Setup}

Since this system has three major parts (Web Application, Backend API, and Mobile Application), different development tools and frameworks are used for each part.

This project source code is maintain and hosted on GitHub for version control and collaboration with team members. The web application is deployed on Vercel and the mobile app on the Play Store; we used all technology used in this project are free and open source.

\subsection{Development Architecture}
The Pharmaciano system follows a modern \textbf{three-tier architecture}.
\begin{enumerate}
\item Presentation Layer
\begin{itemize}
\item Web application developed using Next.js.
\item Mobile application developed using Flutter.
\end{itemize}

\item Business Logic Layer
\begin{itemize}
\item Express.js REST API.
\item Authentication and authorization.
\item Business logic for inventory, sales, purchases, warehouse, accounts, AI prediction, and reports.
\end{itemize}

\item Data Layer
\begin{itemize}
\item MongoDB stores all business data.
\item Redis stores cache data for faster response.
\item JWT is used for secure authentication.
\end{itemize}
\end{enumerate}
This architecture makes the system maintainable, scalable, and reliable for future upgrades.

\subsection{Security Evaluation}
Several security mechanisms are tested to protect the system for unautorized access, data leakage, and wrong changes.

The implemented security features:
\begin{itemize}
\item JWT authentication for secure login.
\item Role-Based Access Control (RBAC) for permission management.
\item Password encryption before storing user credentials.
\item Protected API routes for authenticated users only.
\item Input validation to reduce invalid or malicious requests.
\item Secure communication between frontend and backend using HTTPS after deployment on cloud.
\end{itemize}

These security features help us to protect business data and prevent unauthorized system access.

\subsection{Discussion of Results}

The tested results show that the system performance well for daily pharmacy operations.

The Next.js frontend deliver a fast and responsive user experience. The Express.js backend processes requests efficiently, while MongoDB stores business data in a flexible way. Redis improves system performance by reducing repeated AI database queries.

The implemented Role-Based Access Control (RBAC) and Permission Based Access Control (PBAC) allows the Super Admin to create custom roles and assign permissions based on organizational requirements. This makes the system suitable for both small and large pharmacy businesses.

The multi-branch architecture provides each branch to manage its own inventory, warehouses, customers, purchases, and sales independently, while medicine information remains centrally managed. This helps businesses to maintain data consistency for all branches.

Another important feature is the integration of the Google Gemini API. The AI analyzes historical sales data and predicts future medicine demand. This helps pharmacy owners make better purchasing decisions and reduce stock shortages or overstock situations.

Overall, the developed technologies worked together successfully and fulfilled the project requirements.

\subsection{Advantages of the Developed System}
The developed system delivers several benefits as compared to traditional manual pharmacy management.

Some major advantages are:
\begin{itemize}
\item Automates daily pharmacy operations.
\item Reduces manual calculation errors.
\item Improves inventory management.
\item Supports multiple branches from one system.
\item Provides customizable role and permission management.
\item Generates invoices automatically.
\item Maintains financial records using accounts and ledgers.
\item Predicts future medicine demand using AI.
\item Responsive on desktop, tablet, and mobile devices.
\item Uses only free and open-source technologies, reducing development cost.
\end{itemize}

\subsection{Limitations}
Although the system achieved its objectives, but some limitations still exist in this system.
\begin{itemize}
\item AI prediction depends on historical sales data quality.
\item Google Gemini API requires an internet connection.
\item The free deployment platform has resource limitations.
\item Payment gateway integration is not included in the current version.
\item Nation wide medicine database synchronization not implemented
\end{itemize}

These limitations can be improved in future versions.

These improvements will make Pharmaciano more powerful and suitable for larger pharmacy organizations.
